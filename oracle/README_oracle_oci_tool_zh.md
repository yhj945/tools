# OCI 实例配置更新脚本

一个交互式 Bash 脚本，用于通过 CLI 管理 OCI 计算实例配置。

## 功能特性

- **OCI 环境检查**：验证 OCI CLI、jq、配置文件和连接状态
- **实例管理**：列出、查看、启动、停止实例
- **配置更新**：更新实例 OCPU 和内存设置
  - 直接更新（不停止实例）
  - 完整更新流程（停止→更新→启动）
- **后台任务**：创建和管理后台更新任务，支持自动重试
- **邮件通知**：更新成功后发送邮件通知（SMTP）
- **配置文件支持**：使用 JSON 配置文件批量更新
- **任务恢复**：恢复已停止的任务，保留执行次数

## 系统要求

- Bash 3.x+（兼容 macOS）
- [OCI CLI](https://docs.oracle.com/en-us/iaas/Content/API/Concepts/cliconcepts.htm)
- [jq](https://stedolan.github.io/jq/)
- curl（用于邮件通知）

## 快速开始

```bash
# 运行脚本
./oracle_oci_tool.sh
```

## 主菜单

| 选项 | 描述 |
|------|------|
| 1 | 检查 OCI 环境 |
| 2 | 初始化 OCI 配置 |
| 3 | 查看 OCI 配置 |
| 4 | 管理实例 |
| 5 | 管理后台任务 |
| 6 | 配置邮件通知 |
| h | 帮助信息 |
| 0 | 退出 |

## 配置说明

### OCI CLI 配置

1. 登录 [OCI 控制台](https://cloud.oracle.com)
2. 进入 用户设置 → API 密钥
3. 添加或查看 API 密钥，获取：
   - 用户 OCID
   - 指纹
   - 租户 OCID
4. 下载或创建私钥文件（如 `~/.oci/oci_api_key.pem`）

### 邮件通知（可选）

通过菜单选项 `[6] 配置邮件通知` 配置：
- SMTP 服务器（如 `smtp.qq.com`）
- SMTP 端口（SSL 通常为 `465`）
- 发件人邮箱
- SMTP 密码/授权码
- 收件人邮箱

## 文件结构

```
./
├── oracle_oci_tool.sh      # 主脚本
├── email_config.conf       # 邮件配置（自动生成）
├── tasks/                  # 后台任务数据（运行时生成）
├── instance_update.json    # 配置模板（手动/自动生成）
└── retry_update.sh         # 重试脚本（自动生成）
```

## 后台任务

- 任务在后台运行，支持自动重试
- 执行次数在重启后保留
- 支持恢复已停止的任务
- 支持实时查看日志
